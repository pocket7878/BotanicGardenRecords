# 植物園訪問ログ向け SEO コンテンツ設計

## 目的

植物園四季採集帖を、植物園を訪れた思い出を残したい個人のための訪問ログアプリとして、検索エンジンと初見の利用者に明確に伝える。

## 対象読者

- 植物園が好きで、訪れた記録を残したい人
- 同じ植物園を季節を変えて訪れることを楽しむ人

四季制覇は主目的ではなく、思い出を積み重ねる利用の延長にある楽しみとして扱う。

## 変更内容

- HTML title を `植物園四季採集帖｜植物園の訪問記録を残すアプリ` にする。
- meta description に、個人用・春夏秋冬の記録・ログイン不要・ブラウザ保存を明記する。
- 既存の導入文を、訪問した季節と思い出を残す体験に置き換える。
- 植物園一覧の前に、標本帳のように記録を重ねる価値を説明するセクションを追加する。
- 使い方を 3 項目で追加する。
- 保存場所、機種変更、四季制覇の強制がないことを答える FAQ を追加する。
- 既存の `assets/botanical-background.png` を背景に、サービス名を正確に重ねた 1200×630px の OGP 画像を作成する。
- OGP 画像、OGP title、OGP description、OGP URL をトップページに設定する。

## 文言

### meta description

`訪れた植物園を春・夏・秋・冬ごとに記録する、個人用の標本帳アプリです。ログイン不要。思い出を残しながら、好きな植物園を四季で巡れます。`

### 導入文

`訪れた植物園を、春・夏・秋・冬の季節ごとに記録する標本帳です。 その日の花や空気、また行きたいと思った気持ちを、あなた自身の採集帖に残していけます。`

### 標本帳セクション

見出し: `訪れた季節を、あなたの標本帳に。`

本文: `行った植物園で、訪れた季節の印をひとつ記録します。 同じ場所でも、春の芽吹き、夏の濃い緑、秋の実り、冬の静けさはそれぞれ違うもの。 少しずつ印を重ねていくと、自分だけの植物園の思い出ができあがります。 気に入った園を、いつか四季すべてで巡る楽しみもここから始まります。`

### 使い方

1. `気になる植物園を、地域や名前で探す`
2. `訪れた季節の印を付ける`
3. `記録はブラウザ内に保存。必要ならJSONでバックアップする`

### FAQ

- 記録はどこに保存されますか？: `このブラウザ内に保存されます。ログインやアカウント作成は不要です。`
- 機種変更やブラウザを変えた場合は？: `JSONで記録を書き出しておけば、別のブラウザで読み込めます。`
- 四季すべてを記録しなければいけませんか？: `いいえ。好きな季節だけでも使えます。何度も訪れた園の思い出を、あなたのペースで残せます。`

## 制約

- 現行の静的 HTML/CSS/JavaScript と localStorage 保存を維持する。
- 新しい外部サービス、ログイン、解析コード、広告は追加しない。
- OGP 画像に新しい生成画像は使わず、既存の植物図鑑風背景を再利用する。
- コンテンツは植物園の訪問ログを主題にし、一般的な植物園ガイドにはしない。
- commit と push は行わず、差分とローカル表示を利用者に確認してもらう。
